Unlimited Jdk

AES Jruby Java UnlimitedStrengthCrypto

今天在开发项目过程中遇到一个JDK加密算法无法使用的问题

环境: Jruby + Oracle Java 1.7 + Rails 3.2.8

在使用 Ruby Api OpenSSL 做AES加密过程中总是报错,永远看到Java的类相关问题. 最初以为是Jdk没有加密工具类,但是相关类加载没有任何问题,例如类:javax.crypto.JceSecurity. 最终,在 Jruby Github 的Wiki上找到解释,原来就是官方的Java(Jdk/Jre)不支持导致的问题。 下面附上链接以及链接说提出的解决办法:

https://github.com/jruby/jruby/wiki/UnlimitedStrengthCrypto

通过链接中的信息可以看到解决办法,原因是Java 类库的限制,因此使用Ruby 动态编程,打开限制即可（但是这种方法也有局限性）

security_class = java.lang.Class.for_name('javax.crypto.JceSecurity')
restricted_field = security_class.get_declared_field('isRestricted')
restricted_field.accessible = true
restricted_field.set nil, false

这种动态修改私有属性值,达到放开限制的目的,然而只适用于Java 1.7在Java 1.8中不能达到效果, 经过查看Java1.8的 javax.crypto.JceSecurity 类的源码发现isRestricted变量的类型是private static final 的（不是所有的Java1.8版本都是final 遇到过Windows环境的jdk1.8.0_91也是private static 的变量）, 而Java 1.7 则是 private static 的。因此上述方法无法奏效（ 不清楚是否所有的Jdk7都是这种情况,未查证）。

那为什么呢？难道private static final 类型的变量就不能动态修改值了吗？ 经过相关查资料找到原因应该是修改了的变量值并不会被方法 isRestricted() 引用了。 因为Java 编译器在生成class文件过程中,对简单类型(int boolean)的 static final变量 进行了编译期优化,优化效果是直接替换了引用处的变量为对应的常量值,这导致,变量修改了引用却无法修改。 自然修改变量值没有效果了,因此在Jdk1.8的情况,只能修改Jdk环境的相关Jar文件了。在Oracle官网 提供了相关环境包的替换修改数据。

链接如下:

http://www.oracle.com/technetwork/java/javase/downloads/jce-7-download-432124.html

http://www.oracle.com/technetwork/java/javase/downloads/jce8-download-2133166.html

出现这种情况的原因:

Note that this circumvents the retrictions intended to comply with US export laws, and you may be in violation of those laws if you are from a country in which the US government intends to restrict access to cryptography technologies.

似乎是国家法律对知识产权的保护。

tips: 所有信息在Jruby官方说明可以了解全面。地址

https://github.com/jruby/jruby/wiki/UnlimitedStrengthCrypto

对于上面提到的Java编译器优化的问题,列出简单例子:

Java源码编译前:

public class StaticOptimize {

    private static final String value = "value";
    private static final boolean isTrue = true;

    static void show() {
        System.out.println(value);
        System.out.println(isTrue);
    }
}

Java代码编译上述代码再反编译结果:

public class StaticOptimize {
    private static final String value = "value";
    private static final boolean isTrue = true;

    public StaticOptimize() {
    }

    static void show() {
        System.out.println("value");
        System.out.println(true);
    }
}

可以看到引用处的值已经被编译成了常量字符串 “value” 和常量 true
