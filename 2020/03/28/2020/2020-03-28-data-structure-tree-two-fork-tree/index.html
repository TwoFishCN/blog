树 - 二叉树

一些定义

二叉树是每个结点最多有两个子树的树结构。通常子树被称作“左子树”（left subtree）和“右子树”（right subtree）。

一棵深度为k，且有2^k-1个结点的二叉树，称为满二叉树。这种树的特点是每一层上的结点数都是最大结点数。而在一棵二叉树中， 除最后一层外，若其余层都是满的，并且或者最后一层是满的，或者是在右边缺少连续若干结点，则此二叉树为完全二叉树。 具有n个结点的完全二叉树的深度为floor()+1。深度为k的完全二叉树，至少有 2^(k-1) 个叶子结点，至多有2^k-1个结点。

若设二叉树的高度为h，除第 h 层外，其它各层 (1～h-1) 的结点数都达到最大个数，第h层有叶子结点，并且叶子结点都是从左到右依次排布， 这样的就是完全二叉树。

在查找二叉树中，我们追求平衡二叉树，这样可以优化查找性能，使得查找开销最优化。

满二叉树的一些性质：

  • (1) 如果一颗树深度为h，最大层数为k，且深度与最大层数相同，即k=h;
  • (2) 它的叶子数是: 2^(h-1)
  • (3) 第k层的结点数是: 2^(k-1)
  • (4) 总结点数是: 2^k-1 (2的k次方减一)
  • (5) 总节点数一定是奇数。
  • (6) 树高: h=。

完全二叉树的一些性质：

  • (1) 深度为k的完全二叉树，至少有2^(k-1)个节点，至多有2^k-1个节点。
  • (2) 树高 h= + 1

存储结构

在程序实现的层面来看，二叉树的存储方式一般有顺序存储、链式存储。两种存储方式都广泛被使用。

顺序存储使用一个连续存储空间来存储节点的数据和关系，在数据操作的插入等操作时大多会产生较高的复杂度。 而链式存储则是将节点看作一个对象，每一个对象在存储自己属性的同时存储子节点的指针，或者引用。 在我的印象里大多数的程序设计实现时会采用链式结构，当然顺序存储也是存储一些应用场景的。

链式结构示意图:

二叉查找树

二叉查找树（Binary Search Tree），又称二叉排序树（Binary Sort Tree），亦称二叉搜索树。

定义

一棵空树，或者是具有下列性质的二叉树： （1）若左子树不空，则左子树上所有结点的值均小于它的根结点的值； （2）若右子树不空，则右子树上所有结点的值均大于它的根结点的值； （3）左、右子树也分别为二叉排序树； （4）没有键值相等的结点。

实现

实现部分主要是排序二叉树的构建和使用。构建部分包括插入节点，删除节点，使用则主要是基于树的性质遍历查找数据节点。

  • 查找: 查寻搜索过程是首先比较值大小，然后根据规则继续搜索左子树或右子树。

  • 构建: 构建树既插入节点到树，步骤是先搜索，如果到达叶子节点根据规则接入左或右子节点。

  • 删除：删除节点时有几个情况需要考虑：

  1. 被删除的节点是叶子节点
  2. 被删除的节点仅仅有左孩子节点或右孩子节点
  3. 被删除的有两个孩子节点

平衡二叉树

根据儿二叉查找树的定义，在构建树的时候插入顺序非常重要，否者树的结构很可能会变成链表结构。这样一来，查寻的时候无异于 数据全遍历。

graph TD;
    A-->B;
    B-->C;

通过在对数据操作的改进，我们可以保证树不会退化为链表，具体方法有，AVL，红黑树，Treap(Tree+Heap)等等。

AVL 会在树节点插入和删除操作时通过 AVL旋转来保证树的高度平衡。 红黑树，Treap(Tree+Heap) 算法的处理手法有所不同，但目的相同—保证树的高度稳定。通过保证树的高度稳定，从而保证查找过程的高效。 但是相应的在插入和删除节点时也带来了额外的开销。

总结

二叉树作为树结构中比较简单易理解易使用的数据结构有很多值得思考的地方。特别是各种状态下表现出来的各种性质，加以利用 就可以在不同的问题下发挥其“数据结构”所带来的性能提升。
