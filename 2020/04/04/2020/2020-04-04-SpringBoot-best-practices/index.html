自动配置 - Auto-configuration

Spring Auto-configuration 会尝试自动的配置我们的Spring Boot 应用，当我们引入了相关Jar包时，Auto-configuration 就会开始生效了。 例如，HSQLDB 依赖在我们的应用中时，如果我们没有手动配置数据库连接Bean，那么Auto-configuration默认会启用一个内存模式的数据库。

要在应用中使用Auto-configuration我们的应用必须包含 @EnableAutoConfiguration 或者 @SpringBootApplication 在我们的 @Configuration 类中。

TIPS: Spring Boot 推荐在整个应用中仅包含 @EnableAutoConfiguration 或者 @SpringBootApplication中的任一个，注意是一个，所以不要 两个都包括了。

Using the @SpringBootApplication Annotation

许多Spring Boot开发人员喜欢他们的应用程序使用自动配置，组件扫描，并能够在其“应用程序类”上定义额外的配置。一个 @SpringBootApplication 注解可以启用下面三项功能。

  • @EnableAutoConfiguration: enable Spring Boot’s auto-configuration mechanism
  • @ComponentScan: enable @Component scan on the package where the application is located (see the best practices)
  • @Configuration: allow to register extra beans in the context or import additional configuration classes

启动自动配置，允许包扫描，允许注册bean或者导入额外的配置类。

TIPS: @SpringBootApplication 也提供了一些别名去自定义 @EnableAutoConfiguration 和 @ComponentScan的配置。

当然以上这几项功能并非强制绑定要一起使用，你也可以定制化的只是用其中一些部分，或启用一些部分。
