Spring Boot With Mybatis

简介

总体上来看, Mybatis 生态提供了很多功能, 已经基本完善可以完全作为DAO层的解决方案使用了。 下面展示具体的操作细节。

项目环境搭建

提供一个完整的 项目示例 ，建议结合代码来了解使用方法。

  • 搭建Spring Boot项目

Spring Boot的项目构建建议使用 spring.io提供的生成工具，使用Maven绿色脚本。

  • 引入Mybatis相关依赖

provided 部分的依赖是为了后续代码生成插件需要的额外依赖项。

  • 配置Mapper代码生成插件
  • 配置代码生成策略

代码生成策略中 targetRuntime="MyBatis3Simple" MyBatis3Simple 会是生成的代码非常简单，所有的方法映射使用tk.mapper完成。

配置细节提要

generatorConfiguration 里面配置了数据库连接信息。

Mybatis的代码生成策略是根据数据库表的结构信息去生成对应的 Domain 类，因此建表需要走在前面。如果你使用的Flyway这类工具， 整个环节会更加契合。

生成的代码分主要有三个部分

  • Model对象
  • Mapper接口

注意Mapper接口的继承关系，tk.mybatis.mapper.common.Mapper 提供了大量操作接口，提供了通用的操作方法。

  • Mapper xml 配置

使用了 tk.mybatis.mapper.common.Mapper 默认 mapper.xml 就简化很多了。没有特殊需求，完全可以直接使用。
